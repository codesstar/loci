# 架构 — 三层记忆系统

## 概览

Loci 将 AI 的上下文组织为三个层次，灵感源自人类记忆的运作方式：工作记忆（始终活跃）、情景记忆（按需唤起）和长期存储（归档备查）。

## Layer 1 — 始终加载

这些文件在每次对话开始时被读取，定义了你是谁以及当前最重要的事。

| 文件 | 用途 |
|------|---------|
| `CLAUDE.md` | 系统规则、行为协议、目录地图 |
| `plan.md` | 人生方向、年度目标、当前重点 |
| `tasks/daily/YYYY-MM-DD.md` | 今日日程、优先级、精力状态 |
| `inbox.md` | 快速捕捉、待处理项 |
| `.claude/memory/MEMORY.md` | AI 的持久化自动记忆 |

**设计原则**：Layer 1 必须保持精简。如果某个文件超过约 200 行，应将详细内容提取到 Layer 2，仅在 Layer 1 保留索引。

## Layer 2 — 按需加载

当对话进入特定领域时，这些文件才会被读取。

| 触发条件 | 加载的文件 |
|---------|-------------|
| 讨论任务 | `tasks/active.md`、`tasks/README.md` |
| 提到某个人 | `people/person-name.md` |
| 做计划 | `tasks/daily/YYYY-MM-DD.md`、模块 README |
| 财务讨论 | `finance/overview.md` |
| 内容创作 | `content/platforms.md` |

**设计原则**：模块的 README 文件充当每个领域的"索引"。AI 先读 README 了解有哪些可用内容，再按需加载具体文件。

## Layer 3 — 深度存储

永远不会自动加载，只在明确需要时才访问。

- `archive/` — 已完成的任务、过期的计划、旧内容
- `decisions/` — 历史决策记录
- `me/evolution.md` — 个人成长时间线
- 详细财务记录、旧日记

**设计原则**：Layer 3 可以无限增长而不影响性能。它是你的可搜索档案库。

## 信息流

```
对话
    ↓
[蒸馏]
    ↓
┌─────────────────────────────────────────┐
│  Layer 1（始终活跃）                     │
│  CLAUDE.md → plan.md → inbox.md         │
│  auto-memory                            │
├─────────────────────────────────────────┤
│  Layer 2（按需加载）                     │
│  me/ → tasks/ → people/                │
│  content/ → finance/                    │
├─────────────────────────────────────────┤
│  Layer 3（深度存储）                     │
│  archive/ → decisions/                  │
│  me/evolution.md                        │
└─────────────────────────────────────────┘
```

## 为什么是三层？

AI 的上下文窗口是有限的。每次都加载所有内容会浪费 token 并稀释注意力。三层系统确保：

1. **速度**：Layer 1 即时加载，让 AI 立刻获得上下文
2. **相关性**：Layer 2 只加载当前话题所需的内容
3. **完整性**：Layer 3 确保没有任何信息会丢失

这与人脑的工作方式如出一辙——你不会同时有意识地回忆所有记忆，但当需要时，一切都是可触达的。

## 设计哲学：Loci 是一个记忆调度器

**Loci 的核心职责是决定 AI 现在应该记住什么——以及可以安全遗忘什么。**

这正是人脑的运作方式：

| 人脑 | 容量 | Loci 对应 |
|-------------|----------|-----------------|
| 工作记忆 | 4-7 个块（非常小!） | Layer 1 — 始终加载，必须保持精简 |
| 短期记忆 | 按需回忆 | Layer 2 — 一步之遥 |
| 长期记忆 | 无限存储 | Layer 3 — 归档，可搜索 |

人脑的海马体充当调度员——它决定哪些记忆提升到工作记忆，哪些整合到长期存储。**Loci 为你的 AI 扮演的正是这个角色。**

这也是竞品不足之处：
- **ChatGPT Memory**：扁平列表，没有分层，没有调度——就像所有记忆以相同音量同时播放
- **Mem0**：有存储和检索，但没有智能调度——就像一个没有图书管理员的档案柜
- **Loci**：**分层存储 + 按需加载 + 主动遗忘**——这三者才是让记忆系统真正有效的关键

> 名字不是巧合。海马体（我们的海马吉祥物）是大脑的记忆调度器。Loci 就是 AI 的记忆调度器。

## 上下文感知

Loci 包含状态感知和跨终端同步系统。详见 **[上下文感知](context-awareness.md)**，包括：

- **状态感知（Context Aura）** — 三层信号模型，从日计划、时间信号和可选的用户手动覆盖中推断用户状态（精力、位置、日程）
- **跨终端同步（Changelog Protocol）** — 共享的追加式日志，让多个终端了解彼此的文件变更
- **`/status` 和 `/sync` 命令** — 轻量级的用户控制，用于手动覆盖和刷新

## 自定义

你可以通过编辑 `CLAUDE.md` 来调整每个层次的内容归属。目录地图和上下文分层规则部分精确定义了什么文件在什么时候被加载。
