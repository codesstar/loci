# 架构 — 三层记忆系统

## 概览

Loci 把 AI 的上下文分成三层来管理，灵感来自人脑的记忆机制：工作记忆（随时在线）、情景记忆（想起来就调出来）、长期存储（存着不丢就行）。

## Layer 1 — 始终加载

每次对话一开始就会读进来的文件，决定了"你是谁"和"现在最重要的是什么"。

| 文件 | 干什么用的 |
|------|---------|
| `CLAUDE.md` | 系统规则、行为协议、目录地图 |
| `plan.md` | 人生方向、年度目标、当前重点 |
| `tasks/daily/YYYY-MM-DD.md` | 今天的日程、优先级、精力状态 |
| `inbox.md` | 快速记录、待处理的事 |
| `.claude/memory/MEMORY.md` | AI 的持久化自动记忆 |

**原则**：Layer 1 必须精简。哪个文件超过 200 行了，就该把细节挪到 Layer 2，这里只留索引。

## Layer 2 — 按需加载

聊到某个领域的时候才会读取。

| 什么时候触发 | 加载哪些文件 |
|---------|-------------|
| 聊任务 | `tasks/active.md`、`tasks/README.md` |
| 提到某个人 | `people/person-name.md` |
| 做计划 | `tasks/daily/YYYY-MM-DD.md`、模块 README |
| 聊钱 | `finance/overview.md` |
| 搞内容 | `content/platforms.md` |

**原则**：每个模块的 README 就是那个领域的"索引页"。AI 先读 README 看看有什么，再按需加载具体文件。

## Layer 3 — 深度存储

不会自动加载，只有明确需要的时候才去翻。

- `archive/` — 做完的任务、过期的计划、旧内容
- `decisions/` — 历史决策记录
- `me/evolution.md` — 个人成长时间线
- 详细财务记录、旧日记

**原则**：Layer 3 随便长，不影响性能。它就是你的可搜索档案库。

## 信息流

```
对话
    ↓
[蒸馏]
    ↓
┌─────────────────────────────────────────┐
│  Layer 1（始终在线）                     │
│  CLAUDE.md → plan.md → inbox.md         │
│  auto-memory                            │
├─────────────────────────────────────────┤
│  Layer 2（按需调取）                     │
│  me/ → tasks/ → people/                │
│  content/ → finance/                    │
├─────────────────────────────────────────┤
│  Layer 3（深度存储）                     │
│  archive/ → decisions/                  │
│  me/evolution.md                        │
└─────────────────────────────────────────┘
```

## 为什么要分三层？

AI 的 context window 是有限的。每次把所有东西一股脑塞进去，既浪费 token 又分散注意力。三层系统解决的是：

1. **快**：Layer 1 秒加载，AI 立刻进入状态
2. **准**：Layer 2 只加载当前话题需要的东西
3. **全**：Layer 3 保证没有任何信息会丢

跟人脑一个道理——你不会同时想起所有记忆，但需要的时候，什么都能找到。

## 设计哲学：Loci 是一个记忆调度器

**Loci 的核心工作就是决定：AI 现在该记住什么，可以忘掉什么。**

人脑就是这么干的：

| 人脑 | 容量 | Loci 对应 |
|-------------|----------|-----------------|
| 工作记忆 | 4-7 个块（非常小!） | Layer 1 — 始终加载，必须精简 |
| 短期记忆 | 想起来就有 | Layer 2 — 一步之遥 |
| 长期记忆 | 无限 | Layer 3 — 归档，可搜索 |

人脑里的海马体就是调度员——它决定哪些记忆拉到前台，哪些整合进长期存储。**Loci 给 AI 干的就是这个活。**

这也是竞品没做好的地方：
- **ChatGPT Memory**：一个扁平列表，不分层，不调度——相当于所有记忆同时以最大音量播放
- **Mem0**：有存有取，但没有智能调度——就像一个没有管理员的档案柜
- **Loci**：**分层存储 + 按需加载 + 主动遗忘**——三件事一起做，记忆系统才真正好使

> 名字不是巧合。海马体（也是我们的吉祥物）是大脑的记忆调度中枢。Loci 就是 AI 的记忆调度中枢。

## 上下文感知

Loci 内置了状态感知和跨终端同步。详见 **[上下文感知](context-awareness.zh-CN.md)**，包括：

- **状态感知（Context Aura）** — 三层信号模型，从日计划、时间信号和用户手动输入中推断你的状态（精力、位置、日程）
- **跨终端同步（Changelog Protocol）** — 共享的追加式日志，让多个终端知道彼此改了什么文件
- **`/status` 和 `/sync` 命令** — 轻量级的手动控制，用来覆盖状态或刷新同步

## 自定义

直接改 `CLAUDE.md` 就行。目录地图和上下文分层规则那两个部分，精确定义了哪些文件什么时候被加载。你想调整层级归属，改那里就好。
