## 技术专家评审

### 架构评估
- **server.js**: 零依赖设计优秀，约 1100 行，逻辑清晰
- **数据层**: Markdown 文件直接读写，简单但有并发风险（两个同时写入可能冲突）
- **前端**: 单文件 9000+ 行是当前最大技术债

### 安全性
| 项目 | 状态 | 风险 |
|------|------|------|
| API 认证 | ❌ 无 | 低（本地使用） |
| 输入校验 | ⚠️ 部分 | 中（特殊字符在 regex 中可能有问题） |
| 文件路径注入 | ✅ 使用 path.join | 低 |
| XSS | ⚠️ v-html 使用 | 中（任务文本通过 v-html 渲染） |

### 性能
- `/api/data` 每次请求都读取整个文件系统 — 小规模没问题，大量文件时可能慢
- 5 秒轮询而非 WebSocket — 可以接受但不优雅

### 建议
1. **P0**: 修复 v-html XSS 风险（用 sanitizer 或改用文本插值）
2. **P1**: 添加文件写入锁（防并发冲突）
3. **P2**: 考虑文件变更 watch 替代轮询
4. **长期**: 前端拆分为组件
