# Loci 快速上手

> 5 分钟，让你的 AI 开始记住你。

---

## 前置条件

你需要两样东西：

1. **Claude Code** — Loci 专为 Claude Code 打造。如果还没装，[点这里安装](https://docs.anthropic.com/en/docs/claude-code/overview)。
2. **Git** — 你大概率已经有了。跑一下 `git --version` 确认。

可选：
- **Python 3** — 只有 Dashboard 需要用到，其他功能不依赖它。

---

## 安装

### 方式 A：克隆后直接启动（推荐）

```bash
git clone https://github.com/codesstar/loci.git my-brain
cd my-brain
claude
```

Claude 会检测到这是一个全新的 brain，然后引导你完成初始设置。

### 方式 B：使用安装脚本

```bash
git clone https://github.com/codesstar/loci.git my-brain
cd my-brain
bash install.sh
```

安装脚本在启动 Claude 之前会多做几件事：
- 检查 Claude Code 是否已安装
- 断开模板仓库的 git remote（防止你的个人数据被推送到公开仓库）
- 把 slash commands 安装到 `~/.claude/commands/`
- 在 `~/.claude/CLAUDE.md` 中添加全局 brain 感知

两种方式的结果一样：Claude 启动，开始引导对话。

## 第一次对话

Claude 在全新的 brain 中启动时，会问你几个问题：

```
  欢迎使用 Loci！让我来帮你设置 brain。

  1. 你叫什么名字？
  2. 你做什么工作？（开发者 / 设计师 / 创作者 / 学生 / 其他）
  3. 你现在最重要的事情是什么？
  4. 你通常什么时间工作？
  5. 你习惯用哪种语言？（English / 中文 / 混合）
```

如实回答就好——这些信息会用来生成你的初始文件，之后随时可以改。

回答完毕后，Claude 会创建你的初始文件：

```
  搞定了！你的 brain 已经准备好。以下是我创建的文件：

  - me/identity.md       你的基本信息（名字、角色、当前重点）
  - plan.md              你的使命和目标
  - tasks/active.md      根据你说的内容创建的第一个任务
  - .loci/config.yml     设置（工作时间、语言）

  试试告诉我你最近做了什么决策，或者
  继续工作就好——我会记住重要的东西。
```

就这样，设置完成。接下来正常对话就行。

## 了解你的 Brain

设置完成后，你的 brain 目录长这样：

```
my-brain/
├── CLAUDE.md              AI 最先读取的文件。包含所有行为规则。
│                          你可以阅读，但通常不需要手动编辑。
│
├── plan.md                你的使命、目标和当前优先级。
│                          每次对话都会加载。方向变了就更新它。
│
├── inbox.md               快速捕捉。当你提到一个模糊的想法
│                          （"也许我该学学 Rust"），会存到这里。
│                          每周回顾一次。
│
├── me/                    关于你的一切。
│   ├── identity.md        基本信息：名字、职业、城市、当前阶段
│   ├── values.md          你看重什么（在相关时创建）
│   ├── learned.md         经验和洞察（持续增长）
│   └── evolution.md       identity/values 的旧版本（成长记录）
│
├── tasks/                 你的工作。
│   ├── active.md          当前任务，按优先级排列（P0-P3）
│   ├── someday.md         以后可能会做的想法
│   ├── daily/             每天一个文件——日程 + 完成情况
│   └── journal/           每日总结和反思
│
├── decisions/             每个重大决策一个文件。记录你做了什么决定、
│                          为什么、考虑了哪些替代方案。永久可搜索。
│
├── archive/               什么都不会被删除。完成的任务、旧计划、
│                          过期的决策——全在这里。
│
├── .loci/                 系统内部文件。
│   ├── config.yml         你的设置（持久化模式、工作时间）
│   ├── links/             已连接的外部项目
│   ├── hooks/             自动同步脚本
│   ├── dashboard/         可视化面板（HTML + data.json）
│   └── activity-log.md    最近 14 天的文件变更记录
│
└── docs/                  文档（你正在读的就是其中一份）
```

扩展模块会在你需要时自动创建：
- `finance/` — 预算、资产、财务跟踪
- `people/` — 联系人和会议记录
- `content/` — 写作和发布
- `references/` — 文章、书籍、你想记住的链接

## 日常使用流程

### 早上

在任意项目目录（或 brain 文件夹）打开终端，开始对话：

```
> morning
```

你的 AI 已经知道今天是几号、昨天做了什么、今天有什么安排。它会给你一个简报：

```
  早上好！昨天你搞定了 API 分页功能，但基于游标的方案
  在删除记录时有个边界情况。

  今天：
  - P0：修复分页的边界情况（你留了个备注："删除的记录
    导致游标间隙，试试软删除"）
  - P1：Review Kai 的 PR——已经开了 3 天了
  - P2：开始写部署文档

  "重构 auth 中间件"已经在 P1 待了 9 天了——
  今天做掉还是挪到 someday？
```

### 工作中

正常工作就好。当你做决策、发现新东西或提到任务时，AI 会自动保存：

```
> 我们试了服务端渲染，但对我们的场景来说复杂度太高了。
  还是用客户端渲染加一个 loading 骨架屏吧。

  收到——已保存你的渲染方案决策和权衡理由。

> 对了，staging 环境部署挂了。好像是新依赖导致
  Docker 构建失败。

  已添加"修复 staging Docker 构建"到你的任务列表，P1。
```

不需要说"保存这个"，也不需要特殊语法，正常说话就行。

### 收工

准备结束一天时：

```
> 今天就到这了

  不错的一天。今天做了这些：
  - 用软删除修复了分页的边界情况
  - 决定继续使用客户端渲染
  - Review 了 Kai 的 PR（通过，留了 2 条评论）

  明天的优先级：
  - P0：修复 staging 部署
  - P1：部署文档（周五发版前得写完）
  - P1：跟进 Kai 的 PR 评论

  现在 10:45 了——差不多该休息了。明天见。
```

如果你说"总结"或"journal"，AI 会在 `tasks/journal/` 中写一篇每日总结。

## 连接项目

用了 Loci 几天后，你会想把代码项目也连进来。这是"第二周"功能。

### 如何连接项目

在任意项目目录中打开 Claude Code，运行：

```
/loci-link
```

AI 会：
1. 扫描项目（README、package.json、目录结构）
2. 在项目中创建 `.loci/` 文件夹，放入连接文件
3. 在你的 brain 的 `.loci/links/` 中注册该项目

从此以后，你在那个项目里工作时，AI 就知道你的 brain 里有什么——决策、目标、偏好，全都知道。

### 哪些内容会跨项目同步

不是所有东西都同步，那样太吵了。默认情况下：

- **决策**同步到 brain（重要选择应该在所有地方可见）
- **里程碑**同步到 brain（发布的功能、版本）
- **洞察**同步到 brain（学到的模式）
- **调试笔记**留在本地（临时修复不需要带走）
- **进行中的工作**留在本地（未完成的想法留在当前上下文）

可以用 `/loci-settings` 按项目单独配置。

### 示例：跨项目记忆的实际效果

你在项目 A 中：
```
> S3 上传静默失败了。查了半天发现即使配了 bucket policy，
  也必须显式设置 ACL。

  收到——已保存你的 S3 上传修复方案和根因分析。
```

后来，你在项目 B 中：
```
> 我需要加文件上传功能，打算用 S3。

  你上周在项目 A 里调过一个 S3 上传问题——即使配了
  bucket policy，不显式设置 ACL 的话上传会静默失败。

  建议从一开始就把这个配好。
```

## 使用技巧

**保存错了就说"撤销"。** Loci 的自动保存很智能但不完美。如果它理解错了，说"undo"就能撤回上一次保存。

**模糊的想法会单独存放。** 当你提到一些不确定的东西（"也许我该学学视频剪辑"），Loci 会把它存到单独的列表，而不是创建一个任务。每周回顾一下。

**跑 `git log --oneline` 看你的记忆时间线。** 每次保存都是一个 git 可追踪的文件变更。你能看到 AI 什么时候学到了什么。

**用 `/loci-consolidate 7` 做周回顾。** 它会扫描过去 7 天的变更，找出规律、过期的任务和你可能忽略的洞察。

**上下文快满了？保存后重新开始就好。** 当 AI 开始变慢或重复自己时，说"save everything"或跑 `/loci-sync`，然后开一个新终端。AI 会读取你的 brain 文件，几秒钟就能接上之前的进度——不用花几分钟重新解释。

**你的 brain 就是普通文件。** 你可以直接在编辑器里改任何文件。Loci 会在下次对话时读取变更。没有任何锁定。

**Dashboard 可以给你一个可视化概览。** 运行 `cd .loci/dashboard && python3 build.py && python3 -m http.server 8765`，然后在浏览器打开 `localhost:8765`。

## 常见问题

**Q：Loci 会把我的数据发到哪里吗？**
不会。所有数据都是本地 Markdown 文件，存在你自己的电脑上。没有服务器、没有账号、没有遥测。你的对话会经过 Claude Code（它有自己的隐私政策），但 Loci 的记忆文件永远不会离开你的电脑。

**Q：我能在 Cursor / Windsurf / 其他编辑器中使用 Loci 吗？**
部分可以。任何 AI 编辑器都能读取你的记忆文件（它们就是 Markdown）。但完整体验——自动保存、slash commands、跨项目同步——需要 Claude Code。详见 [其他编辑器指南](other-editors.md)。

**Q：如果我同时开了两个终端怎么办？**
Loci 会在每次对话开始时检测其他终端的文件变更。同时写同一个文件可能会冲突，但 git 会追踪一切，所以数据不会真正丢失。实际中这种情况很少见。

**Q：怎么备份我的 brain？**
它就是一个 git 仓库。推送到私有的 GitHub/GitLab 仓库，或者直接复制文件夹。标准的 git 备份方法都适用。

**Q：我能把 brain 搬到另一台电脑吗？**
可以。复制文件夹（或从你的私有远程仓库克隆），运行 `bash install.sh`，一切就会重新连接。安装脚本会在新机器上重建全局感知。

**Q：brain 会变多大？**
日常使用几个月后，预计会有几百个 Markdown 文件，总计几 MB。Loci 会归档旧内容，保持活跃文件精简。Git 历史会更大一些，但这很正常。

**Q：我能删除 brain 里的东西吗？**
可以，但 Loci 更倾向于归档而不是删除。建议把文件移到 `archive/`。如果你确实想彻底删除，删掉文件然后 commit。它在 git 历史中仍然存在，除非你重写历史。

**Q：如果我不喜欢自动保存怎么办？**
切换到手动模式：运行 `/loci-brain-settings`，把 persistence 设为 `manual`。手动模式下，除非你明确说"保存这个"或运行 `/loci-sync`，否则什么都不会保存。

**Q：怎么卸载 Loci？**
删除 `~/.claude/CLAUDE.md` 中的 `<!-- loci:start -->` 区块，删除 `~/.claude/commands/` 下的 slash commands，然后删除你的 brain 文件夹。没有系统级的修改需要还原。

**Q：我发现了 bug / 有功能建议。**
在 [GitHub](https://github.com/codesstar/loci/issues) 上提 issue。欢迎贡献——详见 [CONTRIBUTING.md](../CONTRIBUTING.md)。
