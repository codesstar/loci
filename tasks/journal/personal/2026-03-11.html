Auth Bug Deep Dive

Spent 3 hours hunting down a race condition in the auth token interceptor. The issue: when multiple API calls fire simultaneously and the token expires, each one tries to refresh independently, causing duplicate refresh calls.

Solution: Implemented a queue pattern — first caller refreshes, others wait for the promise.
